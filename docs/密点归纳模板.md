## 2. 密点分类与分级
- High: 泄露后可直接造成资产损失或系统被接管（密钥、私钥、生产口令）。
- Medium: 可能造成信息泄露或权限扩大（弱口令、敏感配置、高危接口缺乏鉴权）。
- Low: 风险较低但应治理（调试信息、非生产凭证、冗余暴露字段）。

## 7. 长效机制
- 提交前扫描: 本地执行 `python3 scripts/security_scan.py --fail-on-high`
- CI 门禁: 在 GitHub Actions 中执行同样命令
- 例外管理: 行内添加 `secretscan:ignore`（必须附注理由并代码评审）
- 周期复核: 每周一次密点归纳复盘

## 五、关键发现

### 5.1 Kronos 整体表现平平，但**极值分位具备显著选股能力**

- 整体方向准确率 **49.79%** ≈ 随机基线 50%，**不能单靠整体信号用于选股**。
- 但把预测按强度切片后，画面完全不同：**预测涨幅 > +2%** 的样本中，方向准确率达到 **H+1 61.5% / H+2 76.7% / H+3 66.0%**，显著高于随机。
- **Top 10% 分位命中**：H+1 **66.44%**、H+2 **71.81%**、H+3 **61.07%**，这是选股场景真正关心的指标，**表明 Kronos 在挑出"最看好"的股票时具备可用性**。

### 5.2 存在系统性低估偏差（重要）

- 预测 0 ~ -2% 区间的方向准确率仅 **35.7% / 47.3% / 37.0%**（远低于 50%），意味着当 Kronos 预测"小跌"时，**实际上股票大概率是上涨的**。
- 这说明 Kronos 对 A 股有**整体低估倾向**，样本数据中"预测看跌但实际上涨"的比例极高。
- 解读：**不要把 Kronos 弱看跌当真**，它可能只是在模拟 random walk 的默认下偏；但**强看涨（>+2%）是有参考价值的**。

### 5.3 H+2 反常优于 H+1

- H+1 准确率 48.76%、H+2 准确率 **54.93%**、H+3 准确率 45.67%。
- H+2 的 IC（0.1219）也是三个窗口里最高的，接近金融量化中的"有意义信号"阈值（≥0.05 即可用，≥0.1 属于有价值）。
- 解读：Kronos 在**中期（2 日）视野上比短期和长期都稳**，短期受噪声干扰、长期误差累积，这符合时序预测的共识。

### 5.4 绝对价格 MAPE 仅 2.26%，但别被迷惑

- 收盘价 MAPE=2.26% 看起来很低，但 A 股日均波动本就 1-3%，**MAPE 低不等于模型强**。
- 真正反映预测能力的是**方向准确率**和 **IC**，而非绝对值偏差。

## 六、给系统的使用建议

### 6.1 三层信号过滤（已在"预测选股"模块中实现）

| 信号强度 | 方向准确率 | 建议用途 |
|---|---|---|
| 预测 > +2%（强看涨） | **H+2 76.7% / H+3 66.0%** | ✅ 直接入买入候选池 |
| 预测 0 ~ +2%（弱看涨） | H+1 66.2% / H+2 64.5% | ✅ 可入重点关注池 |
| 预测 0 ~ -2%（弱看跌） | 35-47% | ❌ **不可信**，低估偏差区间 |
| 预测 < -2%（强看跌） | H+2 57.3% / H+3 38.5% | ⚠️ 仅 H+2 有参考价值，止损信号用 |

### 6.2 策略决策落地

1. **只看多，不看空**：Kronos 对看跌信号不可靠，弱看跌区间甚至反向。系统应**只用预测涨幅 > +2% 的信号做入池**。
2. **用 H+2 的 Top 10% 作为核心入池源**：71.81% 命中率已经达到可用水平，这是所有维度里最强的信号。
3. **与概念板块、公告关键词多因子合成**：单一 IC 0.041 太弱，必须组合。当前系统的"预测选股"模块已经做了「概念板块热门 Top 10 × 板块领涨 Top 10 × Kronos 预测」的三级过滤，把随机样本 49% 的基准提升到了分位命中 66%+ 的区间。
4. **不要使用预测的绝对价格下单**：价格带噪声，**只取方向**。实盘模块应对预测价格做 ±5% 容忍带或用 OCO 订单。
5. **定期重评**：模型有漂移风险，建议每月重跑 `scripts/evaluate_kronos.py` 追踪准确率变化，且扩展 lookback 到 60-120 日（需先扩大 DB K 线缓存范围）。

### 6.3 本报告的局限

- **样本量**：497 股 / 5019 股（10% 抽样），完整评估可能会把整体准确率收敛到 50±2%，**但极值分位的结论方向不会变**。
- **Lookback=20**：受限于本地 K 线库只缓存 30 天，真实场景 Kronos 推荐 60-120 日上下文。建议把 K 线库扩展到 120 天后重跑。
- **锚点集中在最近 3 日**：不代表全年分布，行情风格切换期（牛熊转换）准确率会波动。

# 热门股票智能分析模块

## 背景 / 目标

当前系统已有“热门个股 Top30”展示，但缺少对热门股票的批量智能分析、评分排序与分池能力。本次目标是基于热门股票前 20 名，逐个生成结构化分析结果、输出综合评分，并按候选池 / 重点关注池 / 买入池三池策略展示到页面模块中。

## 任务分解（checklist）

- [ ] 梳理热门股票数据源、Kronos 预测能力和现有三池模块复用点
- [ ] 实现热门股票智能分析服务：拉取 Top20、逐只分析、综合评分、三池分层、快照持久化
- [ ] 新增 FastAPI 接口：读取快照、手动触发分析、读取/更新配置
- [ ] 新增前端模块 / Tab：展示分析进度、三池卡片、单股详情联动
- [ ] 补充测试与 README 文档，完成服务重启、验证、提交推送

## 验收标准

- 支持对热门股票前 20 名批量分析，并返回可解释的结构化评分结果
- 页面中能查看候选池 / 重点关注池 / 买入池三池结果
- 支持手动触发刷新，能展示运行进度和最近一次快照
- 至少有基础自动化测试覆盖评分或分池逻辑
- 修改完成后完成 `./restart.sh`、验证接口可用、提交并推送到 `origin/main`

## 风险与回滚

- 批量调用 Kronos 可能较慢，需限制分析股票数并复用现有轮询逻辑
- 若实时数据缺失，需用 K 线缓存兜底，避免整批任务失败
- 若评分维度不足，不引入高成本 LLM 批量调用，先用结构化规则保证稳定性

## 关键决策记录

- 默认对热门股票 Top20 做结构化智能分析，不对 20 只股票逐一发起 LLM 调用
- 优先复用现有 `KronosPredictService`、`SQLiteStateStore`、三池前端样式与详情弹窗

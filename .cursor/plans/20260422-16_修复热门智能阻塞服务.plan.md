# 修复热门智能阻塞服务

## 背景 / 目标

当前 `热门股票智能分析` 在自动刷新时会触发 Top20 逐股分析，并默认串行接入 `TradingAgents` 讨论。实际运行中，服务端口仍存活，但首页与部分接口会出现长时间无响应，说明自动任务对主服务可用性造成了影响。

本次目标：

- 修复自动刷新导致页面与接口卡死的问题。
- 保留手动分析能力与三池结果展示。
- 降低后台自动任务的资源占用，避免影响主站访问。

## 任务分解（checklist）

- [ ] 梳理 `hot_stock_ai` 调度链路，确认阻塞点与自动刷新触发路径。
- [ ] 调整自动刷新执行方式，避免在调度循环中直接长时间占用主流程。
- [ ] 为自动任务增加轻量模式，默认跳过高耗时的 `TradingAgents` 讨论或降级分析范围。
- [ ] 补充后端测试，覆盖自动任务配置与轻量执行行为。
- [ ] 如页面文案或配置含义变化，同步更新 `README.md`。
- [ ] 执行 `./restart.sh`，验证首页与关键接口恢复可访问。
- [ ] 完成 `git add` / `git commit` / `git push origin main`。

## 验收标准

- 首页 `/` 能稳定返回，不因 `hot_stock_ai` 自动任务而超时。
- `/api/strategy/hot-stock-ai` 与 `/api/strategy/hot-stock-ai/config` 在自动刷新期间也能正常响应。
- 手动触发热门智能分析仍可正常运行并返回三池结果。
- 自动任务不再默认触发完整 Top20 `TradingAgents` 重分析，服务 CPU/响应时间明显改善。

## 风险与回滚

- 若自动任务降级过度，后台结果的新鲜度会下降。
- 若轻量模式与手动模式配置混用不当，可能导致页面展示与用户预期不一致。
- 回滚方式：恢复 `hot_stock_ai` 调度与配置逻辑到本次提交前版本，并重启服务。

## 关键决策记录

- 优先保障主站可用性，自动刷新允许降级，手动分析保留完整能力。
- 不移除 `TradingAgents` 集成，只限制其在自动后台刷新中的默认执行范围。

# 背景 / 目标

数据同步当前远端抓取仍按股票逐只调用 AkShare 单股历史 K 线接口。用户明确要求使用 AkShare 批量接口，一次获取多只股票数据。

目标：在最新交易日增量同步中优先使用 AkShare 全市场批量行情接口，一次获取全市场开高低收/成交量/成交额并批量写入；历史日期或批量接口不可用时再回退单股历史接口。

# 任务分解（checklist）

- [ ] 增加 AkShare 全市场批量行情到 K 线行的转换逻辑。
- [ ] 增量同步最新交易日时优先走批量写入。
- [ ] 保留单股历史接口作为历史日期/批量失败兜底。
- [ ] 更新同步状态与任务日志，明确显示批量接口路径。
- [ ] 更新 README 的数据中心同步说明。
- [ ] 执行 Python/JS/脚本校验。
- [ ] 执行 `./restart.sh` 并验证服务可访问。

# 验收标准

- 最新交易日增量同步优先调用一次 `stock_zh_a_spot_em()`。
- 批量接口成功时，不再对每只股票调用 `stock_zh_a_hist`。
- 批量接口失败或同步历史日期时，仍可回退到原有单股补缺路径。
- 服务重启后数据中心状态接口正常返回。

# 风险与回滚

- 风险：盘中批量行情是实时快照，适合最新交易日；历史日期不能用该接口补。
- 回滚：关闭批量路径，恢复全部使用单股历史接口。

# 关键决策记录

- 批量接口仅用于最新交易日增量补缺。
- 历史全量补缺保留单股历史接口，因为 AkShare 未提供等价全市场历史日线批量接口。

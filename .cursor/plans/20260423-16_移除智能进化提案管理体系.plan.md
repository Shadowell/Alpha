# 背景 / 目标

当前“智能进化”模块同时包含智能监控、AI 能力面板和完整的提案管理体系。提案体系已经覆盖前端 Tab、审批弹窗、FastAPI 接口、Hermes 运行时提案生成、提案学习闭环、MCP 工具和 README 文档。目标是将这套提案体系彻底下线，只保留智能监控、任务执行记录和其他非提案 AI 能力。

# 任务分解（checklist）

- [ ] 移除前端 `提案管理` 子 Tab、提案审批弹窗、提案列表/统计/筛选/批量操作 UI
- [ ] 将手动诊断入口和任务执行记录保留到非提案页面区块
- [ ] 清理前端提案状态管理、事件绑定、接口调用和成功提示文案
- [ ] 删除 FastAPI 中全部 proposal / proposal-learner 路由
- [ ] 删除 Hermes runtime 中提案生成、提案统计、效果追踪和 learner 注入逻辑
- [ ] 删除 Hermes memory 中 proposal / feedback / outcome_tracking 的 schema 初始化与访问方法
- [ ] 删除 MCP server 中 proposal 创建/审批/查询工具
- [ ] 删除不再使用的提案记忆桥接与飞书提案通知实现
- [ ] 更新 README 中智能进化能力、MCP 工具、记忆系统、API 列表和测试说明
- [ ] 更新 API 回归与前端 E2E，用删除后的页面/API 预期替代旧提案用例
- [ ] 执行前端语法检查、Python 编译检查、相关测试并重启服务
- [ ] 提交、推送到 `origin/main`

# 验收标准

- 智能进化页面不再显示 `提案管理` Tab、提案统计卡、提案列表或审批弹窗
- 前端不再请求 `/api/agent/proposals*` 或 `/api/hermes-ai/proposal-learner`
- `/api/agent/proposals*` 与 `/api/hermes-ai/proposal-learner` 不再注册，访问返回 404
- Hermes 定时任务和手动诊断不会创建提案，也不会执行 outcome tracking
- 智能监控、任务记录、风险守门人、自动交易、回测、消息驱动分析、研报、周报功能可正常使用
- `README.md` 中不再出现提案管理、提案学习闭环、提案生命周期相关说明

# 风险与回滚

- 风险：前端残留对 proposal DOM 或接口的引用，会导致 Agent 页报错
- 风险：Hermes runtime 仍然引用已删 memory 方法，会导致定时任务异常
- 风险：README 和测试未同步，造成文档与行为不一致
- 回滚方式：基于本次单独提交整体回退，恢复 proposal 页面、接口和 runtime 逻辑

# 关键决策记录

- 不做数据库删表迁移，历史 `agent_proposals / agent_feedback / agent_outcome_tracking` 表保留为遗留数据
- 手动诊断能力保留，但输出仅为诊断与建议，不再生成提案
- 任务记录保留并挪到监控页内展示，避免因为移除 proposal Tab 丢失运行可见性

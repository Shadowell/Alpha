# 热门智能接入 TradingAgents 讨论流程

## 背景 / 目标

当前热门股票智能分析模块已经具备热门前 20 股票拉取、逐股打分、三池分配与页面展示能力，但讨论输出仍是 Alpha 内部规则与 Kronos 预测结果，未接入 `/Users/jie.feng/work/github/TradingAgents` 的多代理讨论流程。

本次目标是在保留现有实时热门股抓取与三池机制的前提下，将热门智能模块升级为：

- 对热门前 20 股票先做 Alpha 基础打分
- 默认对热门前 20 股票全部调用 TradingAgents 多代理讨论流程，并保留 TopN 配置用于回退
- 将讨论结论转成评分加成与中文摘要
- 在页面上直接展示讨论结论、结论来源与摘要
- 补齐测试、文档、服务重启与代码提交推送

## 任务分解（checklist）

- [ ] 新增 TradingAgents 适配层，统一处理代码路径、A 股代码映射、模型配置与结果抽取
- [ ] 将热门智能服务接入 TradingAgents 讨论流程，并支持缓存、超时、开关与讨论 TopN 配置
- [ ] 在主程序中注入适配器，保证接口启动后即可调用讨论流程
- [ ] 更新热门智能页面，展示讨论评级、评分加成、摘要与来源状态
- [ ] 补充单测，覆盖讨论结果写回评分与缓存旁路逻辑
- [ ] 更新 README，说明热门智能模块已接入 TradingAgents 讨论流程及相关配置
- [ ] 执行编译检查、测试、`./restart.sh`，并完成 `git add/commit/push`

## 验收标准

- 热门智能接口返回结果中包含 `tradingagents` 讨论字段
- 默认前 20 热门股票会触发 TradingAgents 讨论并写回 `decision / summary / score_bonus`
- 页面热门智能卡片可看到讨论结论，不再只有基础分数
- 配置接口可读写 TradingAgents 开关、讨论 TopN、超时时间与模型信息
- 单测通过，服务重启后接口可正常返回数据
- README 中已同步体现新能力与接口说明

## 风险与回滚

- TradingAgents 依赖较重，首次调用可能较慢；通过缓存、超时与可配置 TopN 控制降低影响
- 外部 LLM 或数据依赖异常时，热门智能模块必须回退到 Alpha 基础评分，不允许整体失败
- 若线上表现不稳定，可通过配置关闭 `tradingagents_enabled` 快速回退到原有逻辑

## 关键决策记录

- 继续沿用热门智能现有模块，不新增第二套“热门讨论”模块
- 默认对 Top20 全量讨论，保留 TopN 配置作为性能回退阀门
- 讨论结果只作为评分增强与解释增强，不直接替代 Alpha 的原始基础评分

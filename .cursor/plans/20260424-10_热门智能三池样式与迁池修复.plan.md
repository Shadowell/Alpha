# 背景 / 目标

热门智能页面当前使用独立的 `hot-ai-card` 渲染结构，和策略选股/候选池的 `stock-card` 视觉与交互不一致；候选池卡片也缺少“加入重点关注池”的迁移按钮。页面截图同时显示后端连接失败提示，需要在修复 UI 的同时确认服务端接口可用。

目标：热门智能三池保持与其他候选池一致的卡片风格，并支持从候选池加入重点关注池，同时保留热门智能分析、TradingAgents 评分与详情查看能力。

# 任务分解（checklist）

- [ ] 梳理热门智能前端渲染、三池数据结构和现有普通三池迁池交互。
- [ ] 在后端为热门智能新增安全的三池迁移能力，复用 `MovePoolRequest` 请求结构。
- [ ] 将热门智能卡片改为与普通候选池一致的 `stock-card` 结构，并加入候选到重点关注池的按钮。
- [ ] 补充服务层和 API 回归测试，覆盖热门智能迁池。
- [ ] 更新 README 中热门智能接口与功能说明。
- [ ] 执行 `node --check`、Python 编译/测试、`./restart.sh`，并验证服务接口可访问。
- [ ] 提交并推送到 `origin/main`，避免纳入无关工作区变更。

# 验收标准

- 热门智能候选池卡片样式与其他页面候选池卡片保持一致。
- 候选池卡片可以点击“加入重点”，并移动到重点关注池。
- 重点关注池、买入候选池的已有数据正常渲染，详情点击逻辑不受影响。
- `/api/strategy/hot-stock-ai` 可访问，新增迁池接口返回成功或明确错误。
- 基础校验通过，服务重启后首页可访问。

# 风险与回滚

- 风险：热门智能三池由评分自动分层，手动迁池可能在下一次重新分析后被自动评分结果覆盖。
- 回滚：撤销本次前端卡片渲染调整和新增迁池接口，恢复只读三池展示。

# 关键决策记录

- 本次不改热门智能评分模型和 TradingAgents 分析流程，只补 UI 一致性和手动迁池能力。
- 不新增数据库表，迁池结果写回现有热门智能快照状态。

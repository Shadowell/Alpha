# 数据中心补数 UEs 根因修复计划

## 背景 / 目标

当前数据中心补数卡死的现场证据指向 AkShare 同步网络调用：补数线程卡在 OpenSSL/socket 的阻塞读写，外层 `asyncio.wait_for()` 无法取消 `asyncio.to_thread()` 中已经进入 C 层的 `requests.get(timeout=None)`。目标是让数据中心补数链路不再调用 AkShare 的历史 K、实时 spot、交易日历函数，改为可取消、强超时的 async HTTP 客户端。

同时，18888 / 18889 已被旧 `UEs` 进程占用，本次将脚本默认端口临时切到 18890，并保留 `PORT` 环境变量可覆盖。

- [x] 修改 `KlineCacheService`，数据中心补数路径改用新客户端，不再通过 AkShare `get_hist` / spot / 交易日历。
- [x] 保留现有 API 路径与主要响应结构，补数失败仍按 symbol 记录失败并继续推进。
- [x] 修改 `app/main.py` 注入新客户端。
- [x] 修改 `requirements.txt` 显式声明 `httpx`。
- [x] 修改启动脚本默认端口为 18890，并支持 `PORT` / `PYTHON_BIN` 配置与 Python 架构提示。
- [x] 增加市场数据客户端单测与补数服务回归测试。
- [x] 执行静态检查、相关 pytest、`./restart.sh`，验证 18890 可访问。
- [x] 更新 README，提交并推送到 `origin/main`。

## 验收标准

- 数据中心补数链路不再出现 `stock_zh_a_hist_tx` / `tqdm` 日志。
- `/api/jobs/kline-cache/status` 在补数失败或外部接口超时时仍快速响应。
- 单个 symbol 网络失败只写入失败明细，不阻塞整批任务。
- `./restart.sh` 默认启动 18890，服务首页和数据中心状态接口可访问。
- 相关单测、静态检查通过。

## 风险与回滚

- 直连接口字段若变更，补数字段解析可能返回空；回滚方式是恢复 `KlineCacheService` 到 AkShare 路径，但会重新引入阻塞风险。
- 当前 18888 / 18889 旧 `UEs` 进程只能重启 Mac 清理；本次端口切换只是绕开端口占用。

## 关键决策记录

- 根因修复优先级高于继续兼容 AkShare fallback；数据中心补数宁可失败可重试，也不能无限阻塞。
- 仅替换数据中心补数链路；公告、概念等非高频补数路径暂不纳入本次范围。

# 背景 / 目标

用户要求重启前先断开数据库链接，然后杀掉服务再启动。当前应用已有 shutdown checkpoint，但只在 FastAPI lifespan shutdown 阶段触发；脚本层没有在发送 kill 前主动请求应用释放数据库资源。

目标：新增“重启前准备”流程，`restart.sh` 在停止服务前先请求应用执行数据库 checkpoint/清理，再由脚本侧做 SQLite WAL checkpoint，最后停止并启动固定 18888 服务。

# 验收标准

- 正常服务可响应时，`restart.sh` 先调用 `/api/admin/shutdown-prepare`。
- 应用接口不可响应时，脚本不长时间卡死，继续尝试脚本侧 checkpoint 和停止。
- 服务端口仍固定 18888。

# 风险与回滚

- 风险：当前 18888 若处于 UEs 状态，prepare 请求无法成功，仍需重启 Mac 释放端口。
- 回滚：恢复 `restart.sh` 为直接 `stop.sh && start.sh`。

# 关键决策记录

- 数据库连接以 SQLite 短连接为主，释放动作以 WAL checkpoint + 关闭 checkpoint 连接实现。
- 不在 prepare 接口中直接杀进程，停止动作仍由脚本负责。

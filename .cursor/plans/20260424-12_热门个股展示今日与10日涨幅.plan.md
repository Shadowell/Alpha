# 背景 / 目标

大盘页“热门个股 Top30”当前只展示一个涨幅字段，用户需要同时看到当日涨幅和累计 10 日涨幅，避免把实时涨幅与阶段涨幅混在一起。

目标：`/api/market/hot-stocks` 为每只热门股补充 10 日累计涨幅字段，前端列表明确展示“今日”和“10日”两项涨幅。

# 任务分解（checklist）

- [ ] 在热门股刷新逻辑中基于本地 K 线计算 `cumulative_10d_pct`。
- [ ] 更新 API 数据模型，保证旧快照缺字段时兼容。
- [ ] 更新大盘页热门个股卡片布局，展示最新价、今日涨幅、10 日累计涨幅。
- [ ] 补充测试覆盖 10 日涨幅字段。
- [ ] 更新 README 对热门个股展示的描述。
- [ ] 执行语法检查、测试、服务重启，并提交推送。

# 验收标准

- 热门个股列表中同时展示“今日 xx%”和“10日 xx%”。
- `/api/market/hot-stocks` 每个 item 包含 `cumulative_10d_pct`。
- 本地 K 线不足时 10 日涨幅安全返回 0，不影响热门股列表。
- 页面脚本语法检查通过，相关测试通过。

# 风险与回滚

- 风险：盘中最新价与本地日 K 收盘价存在时间差，10 日累计涨幅会以实时最新价对 10 日前收盘价计算。
- 回滚：撤销 `cumulative_10d_pct` 字段和前端展示，恢复单一涨幅。

# 关键决策记录

- 热门股排序仍按当日涨幅，不改原排行逻辑。
- 10 日累计涨幅作为展示字段，不参与热门智能评分。

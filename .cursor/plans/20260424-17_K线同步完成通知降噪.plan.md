# 背景 / 目标

K 线智能补缺任务每次同步完成都会向飞书推送“同步完成”卡片，数量过多，影响消息流可读性。

目标：同步完成类消息不再推送飞书，只保留失败、异常或明显需要人工关注的通知。

# 任务分解（checklist）

- [ ] 定位 K 线同步完成飞书推送调用点。
- [ ] 移除或跳过成功完成后的 `notify_sync_complete` 调用。
- [ ] 保留日志输出，便于本地排查任务是否完成。
- [ ] 保留失败/异常日志，不新增成功推送。
- [ ] 执行 Python 语法检查。
- [ ] 按项目规则重启服务、提交并推送。

# 验收标准

- K 线智能补缺完成后不再发送飞书“同步完成”卡片。
- 后台日志仍记录同步完成和完整性检查结果。
- 代码编译通过。

# 风险与回滚

- 风险：不再通过飞书看到正常同步完成状态，需要在数据中心或日志查看。
- 回滚：恢复 `notify_sync_complete` 调用。

# 关键决策记录

- 本次只关闭完成类通知，不关闭其他模块飞书推送。
- 不修改 webhook 配置，不影响预测选股或周报推送。

# 背景 / 目标

用户要求服务每次固定使用 18888 端口，不再自动切换到 18892 等备用端口。当前 18888 被旧的 UEs 状态进程占用且健康检查无响应，因此脚本需要明确失败提示，而不是自动换端口。

# 任务分解（checklist）

- [ ] 修改 `start.sh`，固定 `PORT=18888`，不接受环境变量覆盖。
- [ ] 删除自动 fallback 到 18892-18899 的逻辑。
- [ ] 当 18888 被不可用进程占用时，直接报错并提示需要释放端口。
- [ ] 更新 README 启动说明，移除备用端口描述。
- [ ] 执行脚本语法检查。
- [ ] 执行 `./restart.sh` 验证固定端口行为。

# 验收标准

- `./start.sh` 只尝试启动 `18888`。
- 不再出现“改用端口 18892”的输出。
- 如果 18888 被 UEs 旧进程占用，脚本快速失败并给出明确提示。

# 风险与回滚

- 风险：18888 被内核卡死进程占用时，服务无法启动到其他端口。
- 回滚：恢复备用端口 fallback 逻辑。

# 关键决策记录

- 产品要求固定访问入口优先于自动绕开端口冲突。
- 18888 被 UEs 占用时，只能等待系统释放或重启 Mac，脚本不再兜底改端口。

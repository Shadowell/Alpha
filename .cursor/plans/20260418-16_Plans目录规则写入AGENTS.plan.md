# 计划存档规则纳入 AGENTS.md

- **日期**：2026-04-18 16:00 (Asia/Shanghai)
- **触发**：用户要求"每次生产的计划都保存到 `.cursor/plans`，计划名称前面加日期小时，这个写进 AGENTS.md 中"。

## 背景 / 目标

之前的 plans 只会在会话临时出现，关闭窗口即丢失，导致历史方案无法审计和回滚。
需要把"plan 落盘"作为项目级强制规则，所有后续 agent / 对话必须遵守。

## 任务分解（checklist）

- [x] 在 `AGENTS.md` 新增 "计划（Plan）产出规则" 章节
- [x] 约定文件名格式：`YYYYMMDD-HH_<简短主题>.plan.md`（24 小时制，亚洲/上海）
- [x] 约定文件内容必须包含：背景/目标、任务分解、验收标准（推荐再加 风险与回滚、关键决策）
- [x] 约定"只增不删"：同主题更新另起新文件并引用旧文件
- [x] 补齐本次会话已有两个计划的"追溯存档"作为示范：
  - `20260418-16_接入本地Ollama作为Hermes-LLM.plan.md`
  - `20260418-16_Plans目录规则写入AGENTS.plan.md`（本文件）
- [ ] commit + push

## 验收标准

| # | 验收项 | 判定 |
|---|-------|------|
| 1 | `.cursor/plans/` 下文件名均符合 `YYYYMMDD-HH_*.plan.md` 模式 | 必须 |
| 2 | `AGENTS.md` 包含 "## 计划（Plan）产出规则" 章节 | 必须 |
| 3 | 后续每次 agent 生成的"plan / 实施方案"都会落到 `.cursor/plans/` 并遵守命名 | 运行期强制 |

## 关键决策记录

- **小时粒度而非分钟**：一天内同一主题通常不会超过 1–2 次 plan 迭代，小时粒度足够且文件名短；
  真要同小时内多次迭代，可在主题后加后缀如 `20260418-16_主题_v2.plan.md`。
- **时区锁定 Asia/Shanghai**：避免跨时区协作时文件排序混乱。
- **主题用中文**：Alpha 项目 README 和交流都是中文，统一风格。

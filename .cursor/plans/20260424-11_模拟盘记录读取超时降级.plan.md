# 背景 / 目标

模拟盘数据库中的 `paper_positions` 和 `paper_trades` 记录仍存在，但页面显示为空或加载不出。排查发现模拟盘持仓接口在有 open 持仓时会先拉实时行情更新价格；当外部行情接口阻塞时，`/api/paper/positions` 和 `/api/paper/summary` 无法及时返回，前端表现为记录消失。

目标：模拟盘记录查询必须优先返回本地数据库记录，实时行情更新失败或超时时降级，不再阻塞持仓、历史和交易记录展示。

# 任务分解（checklist）

- [ ] 增加模拟盘专用行情快照安全获取方法，限制实时行情等待时间。
- [ ] `positions` / `summary` 接口在行情超时时回退本地快照或直接返回数据库记录。
- [ ] 下单价格获取增加超时保护，避免实时源卡死导致请求悬挂。
- [ ] 补充测试覆盖行情接口超时时仍返回模拟盘持仓。
- [ ] 执行语法检查、模拟盘服务测试和接口验证。
- [ ] 按项目规则重启服务、提交并推送。

# 验收标准

- 数据库存在记录时，模拟盘页面不因行情源异常显示为空。
- `/api/paper/positions` 在行情源阻塞时仍能快速返回本地持仓。
- `/api/paper/summary` 在行情源阻塞时仍能快速返回账户摘要。
- 历史记录和交易流水继续从 `paper_positions` / `paper_trades` 表读取。

# 风险与回滚

- 风险：行情超时时展示的当前价可能不是最新实时价，但优先保证记录可见。
- 回滚：撤销模拟盘行情超时降级 helper，恢复原实时行情优先逻辑。

# 关键决策记录

- 本次不迁移、不清空、不重建模拟盘数据库。
- 模拟盘读接口以“本地记录可见”为第一优先级，价格刷新作为非阻塞增强。

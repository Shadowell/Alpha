# 自定义策略选股重构

## 背景 / 目标

### 背景
当前「策略选股」页面下方的扫描区块是硬编码的**缩量启动 + 首板放量涨停**形态（`QuietBreakoutScanner._evaluate`）：
- 形态逻辑写死在代码里，用户只能调阈值参数（横盘天数/振幅/CV/放量倍数/是否必须涨停）。
- 选出来的结果并不是用户"想要"的形态：用户可能想按"均线多头 + 放量突破前高"、"调整末期缩量 + 下影止跌"、"首板跟进"等完全不同的形态选股，目前无法做到。

上方三池漏斗（调整期候选池 / 重点关注池 / 买入池）**按本期决定保持不动**（用户选 A 方案）。

### 目标
- 把下方「缩量启动」区升级为通用的**自定义策略中心**：
  - 用户可以**新建、编辑、删除、克隆**策略。
  - 每条策略由**若干可组合的原子规则**构成（例如：价格区间、箱体横盘、量能 CV、当日放量、涨停、突破前高、均线多头、排除板块等）。
  - 规则组合**仅支持 AND**（每条规则可开关，关闭的规则直接跳过）—— 本期只做简单模式。
  - 每条策略可**一键在全 A 股历史 K 线上扫描**，输出命中列表 + 规则命中明细 + 综合分。
  - 每条策略可**一键回测**（近 180 天），复用现有 `BacktestLab`，返回胜率/累计收益/最大回撤。
- 内置 **3 条预设策略**：
  1. 缩量启动（等价于现有逻辑，无缝迁移）
  2. 调整期横盘（缩量 + 未破前高 + 下影支撑）
  3. 放量突破前高（均线多头 + 放量 + 突破 N 日高）
- 前端新增**策略中心**面板，与现有三池漏斗并列，可切换查看/编辑/扫描不同策略。
- 现有 `/api/strategy/quiet-breakout` 路由以"兼容重定向 / 适配器"形式保留，避免破坏 MCP / 测试 / E2E 断链。

### 非目标（本期不做）
- 不做**打分模式**（仅 AND）。
- 不做**三池漏斗**的策略化改造。
- 不做**规则间 OR / 括号 / DSL 表达式**这种复杂组合。
- 不做**自定义指标编辑器**（用户写 Python 表达式）—— 只能从预注册规则库里选。
- 不做**多时间窗口**（仅日 K，不含分钟 K / 周 K）。

## 任务分解（checklist）

### 后端 · 规则引擎
- [ ] 新建 `app/services/strategy_rules.py`
  - [ ] 定义 `RuleContext`：封装 symbol、name、kline（日 K 列表）、trade_date、外部查询器（concept map 等）。
  - [ ] 定义 `RuleResult`：`passed: bool`, `label: str`, `metric: float | None`, `detail: str`。
  - [ ] 定义 `RuleSpec`：`code`（唯一标识）, `title`（中文名）, `category`（分类：price / volume / pattern / fundamental / filter）, `params_schema`（每个参数：key / label / type=int|float|bool|list / default / min / max / step）, `evaluate(ctx, params) -> RuleResult`。
  - [ ] 实现初版 10+ 条规则（仅下方单列列出）。
  - [ ] 提供 `RULE_REGISTRY: dict[str, RuleSpec]` 供引擎和 API 调用。
  - [ ] 提供 `list_rules() -> list[dict]` 给前端拉取"规则目录"。

初版规则清单（每条都要可评估 + 必须用到已有 K 线字段，不引入新数据源）：
  - [ ] `price_range` — 最新收盘价 ∈ [min, max]
  - [ ] `market_cap_range` — 流通/总市值区间（从 real-time snapshot 传入；扫描时暂用 `provider.get_realtime_snapshot` 的 `总市值`；若为空 → 跳过该条规则并在结果中标注）
  - [ ] `exclude_boards` — 排除 ST / 创业板 / 科创板 / 北交所（bool 参数 4 个）
  - [ ] `box_consolidation` — 近 N 天箱体振幅 ≤ x%（N / max_amp_pct）
  - [ ] `volume_shrink` — 近 N 天量能 CV ≤ x（N / max_cv）
  - [ ] `volume_spike_today` — 今日量 / 近 N 日均量 ≥ x（N / min_ratio）
  - [ ] `limit_up_today` — 今日涨停（主板 9.7% / 创业科创 19.7% / ST 4.7%；tolerance 容忍值）
  - [ ] `change_pct_today_range` — 今日涨跌幅 ∈ [min%, max%]
  - [ ] `break_prior_high` — 今日收盘 ≥ 过去 N 日最高 × k（N / buffer）
  - [ ] `ma_above` — 收盘 > MA(N)（N）
  - [ ] `ma_bull_stack` — MA 多头排列（ma_list = [5, 10, 20] 三条均线从快到慢递减）
  - [ ] `drawdown_from_high` — 距 N 日最高回撤 ∈ [min%, max%]

### 后端 · 策略模型 & 运行器
- [ ] 新建 `app/services/custom_strategy.py`
  - [ ] `CustomStrategyRule`（dataclass）：`rule_code`, `enabled`, `params: dict`。
  - [ ] `CustomStrategy`（dataclass）：`id`, `name`, `description`, `is_builtin`, `is_default`, `rules: list[CustomStrategyRule]`, `created_at`, `updated_at`。
  - [ ] `CustomStrategyRunner.scan(strategy, *, limit=None) -> ScanResult`
    - 遍历 K 线库所有 A 股代码
    - 对每只股并发执行所有 enabled 规则，全部通过即命中
    - 命中输出：symbol, name, close, prev_close, change_pct, rule_hits=[{code, passed, metric, detail}], composite_score（命中规则数 + 辅助分：放量倍数、箱体紧密度等）
    - 控制并发（Semaphore 16），失败个股静默跳过
  - [ ] 返回 payload 结构与现 quiet-breakout 对齐（`generated_at` / `total_scanned` / `total_hits` / `hits` / `config`），方便前端复用渲染。

## 验收标准

1. 打开「策略选股」tab，下方可见**策略中心**区（替代原缩量启动区），顶部有策略下拉，默认选到"缩量启动"，可看到其规则清单并可修改。
2. 点击"新建策略"→ 起名 → 选 2~3 条规则调参 → 保存 → 下拉中出现新策略，再次刷新页面仍在（SQLite 持久化）。
3. 点击「立即扫描」对任何策略都可返回命中列表，命中卡片显示：名称、代码、涨跌、每条启用规则的命中指标 badge。
4. 内置 3 条策略：**缩量启动 / 调整期横盘 / 放量突破前高** 在 UI 中可见，不能被删除，参数可修改但修改后视为用户版本（仍保留 builtin 标记用于保护删除）。
5. 点击命中卡片仍然可以弹出 Kronos 预测 modal（不触发页面跳转/刷新 / `window.open`）。
6. 「回测」按钮对任意策略返回 `{win_rate, total_return, max_drawdown, total_signals}` 并在 UI 展示。
7. 原 `POST /api/strategy/quiet-breakout/scan` 接口仍能工作并返回结构兼容的响应，`test_api_regression.py` 对应用例通过。
8. `./restart.sh` 后 `logs/server.log` 无 traceback；首页加载 < 3s，策略扫描 < 60s（全 A 股，Semaphore=16）。
9. `README.md` 已同步更新上述变更。

## 风险与回滚

- **风险 1：新规则评估耗时过长。** 每只股要跑多条规则，并发 16 下若规则数到 10+ 可能超过 90s。
  - 缓解：每条规则优先使用轻量向量运算（list/statistics，不走 pandas），早返回（任何一条 fail 即 break）。
  - 回滚：若显著超时，把默认规则限定为 ≤ 6 条 + UI 提示"建议规则数 ≤ 8"。

- **风险 2：前端工作量过大，三池漏斗样式被波及。** 本期严格只替换下方区块 DOM。
  - 缓解：新建独立 `<section class="strategy-center-area">`，原 `quiet-breakout-area` 保留但隐藏（display:none），回滚时只需切换 CSS。

- **风险 3：兼容旧接口可能漏字段。** `QuietBreakoutHit` 的字段（`amp_pct/vol_cv/vol_spike/base_high/base_low/limit_pct/signals/...`）与新通用结构不完全对齐。
  - 缓解：旧接口适配器层做字段映射；对每条命中从规则结果里提取 `metric` 重组成旧结构。
  - 回滚：保留旧 `QuietBreakoutScanner` 实现作为 fallback，只把路由切到新实现。

- **风险 4：builtin 策略每次启动都覆盖用户修改。** 必须以 ID 为主键幂等写入。
  - 缓解：`ensure_builtin_strategies` 实现"只在该 builtin 不存在时插入，存在则跳过"。

- **回滚方案**：
  - 保留 `quiet_breakout_scanner` 实例与旧路由代码路径。
  - 前端把 `strategy-center-area` 的 `display` 切回 `none`，`quiet-breakout-area` 恢复显示即回到原功能。
  - 数据库 `custom_strategies` 表独立，不影响既有 `funnel_state` / `strategy_profiles`。

## 关键决策记录

- **2026-04-18 · 用户拍板**：
  - 范围 A：只重构下方缩量启动区，不动上方三池漏斗。
  - 规则组合 A：仅 AND（每条规则可开关），不做打分模式。
  - 回测 A：每条策略都配套 180 天回测按钮。
  - 预设 A：内置 3 条（缩量启动 / 调整期横盘 / 放量突破前高）。
- **规则注册表（registry）vs 插件化**：选 registry（单 dict）。本期规则数 ≤ 15，简洁优先，不做 entry-point 插件机制。
- **存储模型**：新建 `custom_strategies` 表而非复用 `strategy_profiles`；后者语义是"整个漏斗引擎参数"，与"形态选股规则集"语义不同，混存会带来未来维护成本。
- **兼容接口保留方式**：做适配器而不是直接删除。原因：`test_api_regression.py` 与 MCP tool `trigger_eod_screen` / E2E 都可能依赖；一次性切换风险过大，分两期（本期适配器 + 下期清理）。

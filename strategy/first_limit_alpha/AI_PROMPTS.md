## 1. 总设计提示词

把下面整段直接发给 AI，可以让它先做完整方案设计。

```text
你现在在 /Users/jie.feng/wlb/Alpha 项目中工作，请为一个新的策略模块 `FirstLimit Alpha` 做完整设计与落地实现。

模块目标：
- 面向 A 股短线场景
- 识别“前期震荡后第一次涨停”的股票
- 在首板当天或次日竞价/开盘时给出介入评分
- 预测未来 2-5 个交易日是否继续强势，尤其是继续涨停或显著冲高

重要背景：
- 该模块设计文档在 `/Users/jie.feng/wlb/Alpha/strategy/first_limit_alpha/README.md`
- Alpha 当前已有概念热度、市场情绪、漏斗、Kronos 预测、模拟盘等能力
- 该模块后续应作为 Alpha 的一个独立策略来源，先独立开发，再考虑接入漏斗与前端

核心设计原则：
- 第一版不要直接做“看 K 线图片”的图像深度学习
- 第一版先做结构化特征 + 机器学习 baseline
- 先验证 alpha，再逐步上时序深度学习
- 必须严格避免未来函数
- 必须按交易日滚动切分训练/验证/测试
- 回测要带交易规则，而不是只看分类分数

请输出：
1. 模块的目标定义
2. 标签设计方案
3. 数据源与数据表设计
4. 特征工程分层方案
5. baseline 模型方案
6. 深度学习升级路线
7. 回测与评估框架
8. 如何与 Alpha 现有模块集成
9. 推荐的目录结构和文件拆分
10. 分阶段实施顺序

要求：
- 方案必须能在 Alpha 项目内落地
- 尽量复用现有服务和数据能力
- 不要泛泛而谈，要具体到文件、模块和步骤
- 明确指出哪些信息不能在预测时点使用
```

## 2. 第一阶段：数据与标签工程提示词

这段提示词适合让 AI 先完成样本集和标签定义。

```text
你现在在 /Users/jie.feng/wlb/Alpha 项目中工作，请实现 `FirstLimit Alpha` 的第一阶段：数据与标签工程。

目标：
- 为“前期震荡 + 首板介入 + 后续连板/强势延续”构建训练样本
- 先完成可复现的数据集构建，不做模型训练

请在不破坏现有系统的前提下完成以下工作：

1. 在 `strategy/first_limit_alpha/` 下新增适合的数据构建脚本或模块
2. 明确实现“前期震荡”的数值化定义
3. 明确实现“首板”样本筛选
4. 设计并产出至少三类标签：
   - 连板延续标签
   - 3 日强势标签
   - 断板风险标签
5. 确保预测时点之前不可见的数据不会泄漏
6. 输出一个标准化数据集文件或中间表结构，供后续特征工程使用

建议输出文件：
- `strategy/first_limit_alpha/data_builder.py`
- `strategy/first_limit_alpha/labeling.py`
- `strategy/first_limit_alpha/schema.md` 或代码内数据类定义

必须说明：
- 样本时间窗口
- 标签计算逻辑
- 训练/验证/测试切分建议
- 哪些字段属于未来信息，必须排除

验收要求：
- 能构建一批首板样本
- 每个样本至少包含：股票、交易日、是否首板、标签 A/B/C
- 不允许随机打乱时间切分
```

## 3. 第二阶段：特征工程提示词

```text
你现在在 /Users/jie.feng/wlb/Alpha 项目中工作，请为 `FirstLimit Alpha` 实现第二阶段：特征工程。

目标：
- 基于已构建的首板样本，生成可供机器学习模型训练的结构化特征

请完成：

1. 按以下层次设计特征：
   - 价格行为特征
   - 成交与换手特征
   - 板块与情绪特征
   - 个股属性特征
   - 微结构特征（如果当前数据不足，可先留接口）
   - 派生交互特征
2. 优先复用 Alpha 当前已有能力：
   - ConceptEngine
   - 市场情绪或大盘统计逻辑
   - Kronos 预测能力（只可作为辅助特征，不能喧宾夺主）
3. 为每类特征写出明确计算逻辑
4. 为每个特征标注“预测时点是否可用”
5. 输出特征 DataFrame 或中间持久化文件

建议文件：
- `strategy/first_limit_alpha/features.py`
- `strategy/first_limit_alpha/feature_store.py`

要求：
- 特征命名清晰
- 不要引入未来函数
- 对缺失值和异常值做基础处理
- 把不能稳定获取的微结构特征做成可选层，不要阻塞 baseline

验收要求：
- 至少形成 30+ 个结构化特征
- 特征分层清晰
- 可以用于后续训练 LightGBM/XGBoost baseline
```

## 4. 第三阶段：机器学习 baseline 提示词

```text

要求：
- 不允许随机切分样本

验收要求：
- 跑通 baseline 训练
- 给出测试期的排序效果
```

## 5. 第四阶段：时序深度学习升级提示词

```text
你现在在 /Users/jie.feng/wlb/Alpha 项目中工作，请为 `FirstLimit Alpha` 设计并实现第四阶段：时序深度学习升级。

前提：
- baseline 已经存在
- 目标不是替代 baseline，而是验证时序深度学习是否能带来增量

请完成：

1. 先设计适合金融时间序列的输入，而不是直接喂 K 线截图
2. 候选模型可以包括：
   - LSTM
   - GRU
   - Temporal CNN
   - Transformer for Time Series
3. 输入数据建议包括：
   - 多日 OHLCV
   - 换手率
   - 量比
   - 市场情绪代理变量
   - 板块热度序列
4. 与 baseline 做对比实验
5. 如果效果没有显著增量，要诚实输出结论

建议文件：
- `strategy/first_limit_alpha/sequence_dataset.py`
- `strategy/first_limit_alpha/train_sequence.py`
- `strategy/first_limit_alpha/sequence_model.py`

要求：
- 明确 baseline 对比
- 不要引入过度复杂的图像方案
- 优先验证“序列信息”是否有增量

验收要求：
- 可训练一个时序模型
- 输出和 baseline 的对比报告
- 明确结论：增量是否成立
```

```text
- 回测必须可复现

- 能基于模型输出做 TopK 回测
- 能输出至少一个测试期的完整评估报告
```

## 10. 给 AI 的统一约束

下面这段可以在每次实现前附加给 AI，减少跑偏。

```text
统一约束：

1. 你在 /Users/jie.feng/wlb/Alpha 项目内工作，必须遵守该项目的 AGENTS.md 规则。
2. 本模块名称为 `FirstLimit Alpha`，目录位于 `strategy/first_limit_alpha/`。
3. 第一版优先做结构化特征 + 机器学习 baseline，不要直接做 K 线图像模型。
4. 必须严格避免未来函数，只能使用预测时点之前可见的数据。
5. 训练/验证/测试必须按时间滚动切分，禁止随机切分。
6. 任何模型结果都必须落到交易可执行的回测评估上，不能只给分类分数。
7. 尽量复用 Alpha 已有能力：ConceptEngine、市场情绪、Kronos、模拟盘、漏斗体系。
8. 代码应尽量模块化，方便后续接入服务层和前端。
9. 如果修改了代码、配置或脚本，必须执行 `./restart.sh`。
10. 完成后必须更新 README（如项目结构发生变化），并提交推送到 `origin/main`。
```
